## Data Flow

1. **Capture** (continuous + adaptive)
   - IMU + GPS: Always-on (low cost)
   - Audio + Video: Periodic (every 5-10 min) or triggered
   
2. **Privacy Pipeline** (real-time)
   - Face detection → Blur
   - Text detection → Blur
   - Audio keywords → Filter
   
3. **Understanding** (on-device)
   - Activity classification
   - Scene segmentation
   - Feature extraction
   
4. **Storage** (hierarchical)
   - Short-term: SQLite (24h)
   - Medium-term: Vector DB (30d)
   - Long-term: PostgreSQL + patterns
   
5. **Agent Integration** (on-demand)
   - Retrieve relevant memories
   - Inject into OpenClaw context
   - Personalized response

## Tech Stack

**On-Device:**
- iOS: Swift + CoreML
- Android: Kotlin + TensorFlow Lite
- Privacy: MediaPipe (face/text), Whisper (audio)
- Storage: SQLite

**Cloud:**
- Vector DB: Chroma (dev) / Pinecone (prod)
- Metadata: PostgreSQL
- Backend: Python + FastAPI

**ML Models:**
- Activity: Ego4D baseline models
- Video embeddings: CLIP / VideoCLIP
- Audio embeddings: Whisper / CLAP
- IMU features: Custom LSTM / Transformer

## Constraints Analysis

| Constraint | Target | Feasibility | Strategy |
|-----------|---------|--------------|----------|
| **Battery** | <30%/day | ⚠️ Challenging | Adaptive sampling, duty cycling |
| **Storage** | <2GB/day | ✅ Feasible | Aggressive compression, selective storage |
| **Latency** | <1s | ✅ Feasible | On-device processing, lightweight models |
| **Accuracy** | >75% F1 | 🟡 Unknown (needs testing) | Start with SOTA (Ego4D), optimize if needed |

## Key Challenges

1. **Battery Optimization**
   - IMU + GPS: ~2-4%/day ✅
   - Video + Audio: ~15-20%/day ⚠️
   - **Solution:** Duty cycle (10s capture every 5 min), motion-triggered
   
2. **Storage Compression**
   - Raw video: ~1-2 GB/hour ❌
   - **Solution:** Only store embeddings + keyframes, not full video
   
3. **Multimodal Fusion**
   - Challenge: Different modalities, different time scales
   - **Solution:** Separate encoders + late fusion

### Technical Evaluation
```
⚙️ Feasibility Assessment: [Approach/Component]

**Multimodal Embedding Strategy**

**Options:**
1. **CLIP (Vision-Text)** - Pre-trained, multimodal
   - Pros: SOTA, easy to use
   - Cons: No audio/IMU
   
2. **ImageBind (Meta)** - 6 modalities (inc. IMU)
   - Pros: Comprehensive, SOTA
   - Cons: Large model (~3GB), slow on mobile
   
3. **Separate Encoders + Concat**
   - Video: CLIP, Audio: Whisper, IMU: Custom, Location: Place2Vec
   - Pros: Flexible, optimized per modality
   - Cons: Need fusion logic

**Recommendation:** Separate encoders + early fusion

**Analysis:**
- **Feasibility:** ✅ HIGH - Each encoder well-tested
- **Battery:** ✅ OK - Can use lighter models
- **Storage:** ✅ OK - Embeddings are small (512-768 floats)
- **Accuracy:** ✅ GOOD - State-of-the-art encoders

**Implementation Plan:**
```python
# Pseudo-code
def create_multimodal_embedding(video, audio, imu, location):
    # Encode each modality separately
    video_emb = CLIP_video.encode(video_frame)
    audio_emb = Whisper.encode(audio_segment)
    imu_emb = IMU_encoder.encode(imu_window)
    location_emb = Location_encoder.encode(location)
    
    # Early fusion
    combined = torch.cat([video_emb, audio_emb, imu_emb, location_emb])
    
    # Project to shared space
    embedding = projection_layer(combined)
    
    return embedding  # ~512-768 floats
```

**Tools:**
- Video: CLIP (OpenAI) or MobileCLIP (on-device optimized)
- Audio: Whisper (OpenAI) or CLAP (contrastive)
- IMU: Custom LSTM/Transformer (train on activity data)
- Location: Place2Vec or geohashing

**Estimated Effort:** 1-2 weeks

**Risks:**
- Fusion may not capture cross-modal interactions
- Need to train projection layer
- On-device inference may be slow

**Mitigation:**
- Start with weighted sum (simple), upgrade to learned projection
- Use MobileNet variants for on-device efficiency
- Batch embeddings to reduce overhead
```

  - Free, no credit card needed
  - Easy setup: `pip install chromadb`
  - Python-native, good for local testing
  - Perfect for 4-week user study (N=8-12, ~60GB total)
  
- **Pinecone for Scale:**
  - Managed service (no ops)
  - SOTA performance
  - Production-ready
  - Good if thesis scales beyond study

**Quick Start (Chroma):**
```python
import chromadb

# Initialize
client = chromadb.Client()

# Create collection
collection = client.create_collection(
    name="user_memories",
    metadata={"hnsw:space": "cosine"}
)

# Add memory
collection.add(
    embeddings=multimodal_embedding,  # shape: (1, 512)
    metadatas={
        "type": "activity",
        "timestamp": "2026-02-27T10:30:00Z",
        "duration": 120,
        "location": "Starbucks, Oakland"
    },
    ids="mem_001"
)

# Query relevant memories
results = collection.query(
    query_embeddings=current_context,
    n_results=5,
    where={"type": "activity"},  # optional filter
    where_document={"location": {"$ne": None}}  # only memories with location
)
```
